### Полный функционал бота и протокол для будущих изменений

#### 1. Общая структура
- **Язык и зависимости**:
  - Node.js с модулем `node-telegram-bot-api` для взаимодействия с Telegram API.
  - Зависимости: `fs/promises`, `path`, `luxon` (для работы с датами и временем в `messageScheduler.js` и `logger.js`).
  - Логирование: кастомный `logger.js` перехватывает `console.log`, `console.error`, `console.warn`, `console.info` и отправляет логи в Telegram-чат (`1442060396`) через отдельный бот (`7996684943:AAEqb5txhw8PChJJm4KWpYORLDwT2Ebu12Y`) и в файл `bot.log`.

- **Файлы**:
  - `index.js`: Основной файл, инициализация бота, обработка сообщений, отложенные сообщения, callback-запросы.
  - `adminHandler.js`: Обработка команд администраторов в админском чате (`-1002737100564`).
  - `configManager.js`: Загрузка и сохранение конфигурации (`config.json`).
  - `userBroadcast.js`: Рассылка сообщений пользователям и главному админу.
  - `callbackHandler.js`: Обработка callback-запросов (кнопок: `approve`, `reject`, `approve_with_guide`, `ban`, `to_main_admin`).
  - `messageScheduler.js`: Планирование отложенных сообщений с использованием `setTimeout`.
  - `logger.js`: Логирование в Telegram и файл.

- **Конфигурация (`config.json`)**:
  ```json
  {
    "botToken": "YOUR_BOT_TOKEN",
    "adminChannelId": "-1002737100564",
    "channels": {
      "test": { "channelId": -1002707338601, "name": "Тестовый канал" },
      "TEST": { "channelId": -1002698912911, "name": "ТЕСТ 2" },
      "test3": { "channelId": -1003062939810, "name": "3 канал" }
    },
    "users": {
      "1442060396": { "isAdmin": true, "channelCode": null },
      "5489584315": { "isAdmin": true, "channelCode": "test" }
    },
    "bannedUsers": {},
    "pendingMessages": [],
    "scheduledMessages": []
  }
  ```
  - `botToken`: Токен бота.
  - `adminChannelId`: ID админского чата (`-1002737100564`).
  - `channels`: Список каналов с их ID и названиями.
  - `users`: Пользователи, их статус администратора и привязка к каналам.
  - `bannedUsers`: Забаненные пользователи.
  - `pendingMessages`: Ожидающие проверки сообщения (предложки).
  - `scheduledMessages`: Запланированные сообщения.

#### 2. Основной функционал
##### 2.1. Запуск бота (`index.js`)
- Инициализация:
  - Загружает `config.json` через `loadConfig`.
  - Создаёт экземпляр `TelegramBot` с `polling: true`.
  - Проверяет права бота в админском чате и каналах (`checkBotAdminStatus`).
  - Инициализирует `userState` и `mediaGroups` для хранения состояния пользователей и медиа-групп.
- Логирование:
  - Все логи отправляются в Telegram (`1442060396`) и записываются в `bot.log` через `logger.js`.

##### 2.2. Обработка сообщений (`index.js`, `adminHandler.js`)
- **Пользовательские сообщения**:
  - **Команда `/start`**:
    - Пользователь отправляет `/start <channelCode>` (например, `/start test`).
    - Если `channelCode` существует в `config.channels`, пользователь регистрируется (`config.users[userId] = { isAdmin: false, channelCode }`).
    - Ответ: `Добро пожаловать! Ваши сообщения будут отправлены на проверку для канала <name>.`.
    - Если `channelCode` неверный, ответ: `❌ Пожалуйста, используйте /start с кодом канала, например: /start test`.
  - **Предложка**:
    - Пользователь отправляет текст, фото, видео, документ или медиа-группу.
    - Проверяется, зарегистрирован ли пользователь (`config.users[userId]`), не забанен ли (`config.bannedUsers[userId]`), и есть ли контент.
    - Сообщение сохраняется в `config.pendingMessages`:
      ```json
      {
        "userId": "5489584315",
        "messageId": "275",
        "content": { "text": "Привет, это тест!" },
        "pin": false,
        "channelCode": "test",
        "adminMessageId": null
      }
      ```
    - В админский чат (`-1002737100564`) отправляется:
      - Информационное сообщение: `👤 🆔 <userId> | @username | канал: <channelName>`.
      - Контент (текст/медиа/медиа-группа) с inline-кнопками:
        - `Одобрить`: `approve|<channelCode>|<userId>|<messageId>`
        - `Отклонить`: `reject|<channelCode>|<userId>|<messageId>`
        - `Одобрить с гайдом`: `approve_with_guide|<channelCode>|<userId>|<messageId>`
        - `Забанить`: `ban|<channelCode>|<userId>|<messageId>`
        - `Отправить главному админу`: `to_main_admin|<channelCode>|<userId>|<messageId>`
    - Пользователю: `📩 Ваше сообщение отправлено на проверку`.
    - Для медиа-групп: собираются через `mediaGroups[mediaGroupId]` с таймером 5 секунд, затем отправляются в админский чат.
- **Админские сообщения**:
  - Обрабатываются в `adminHandler.js` в чате `-1002737100564`.
  - Проверяется, что пользователь — администратор чата или имеет `config.users[userId].isAdmin`.
  - Команды (через ReplyKeyboardMarkup):
    - `📅 Создать отложенное сообщение`: Запрашивает контент, каналы и время.
    - `📩 Рассылка пользователям`: Запрашивает контент для рассылки.
    - `📬 Запланировать рассылку`: Запрашивает контент и время для рассылки.
    - `📋 Список каналов`: Показывает каналы (`<name> (<code>)`).
    - `👥 Список пользователей`: Показывает пользователей (`<userId>: <channelCode>, isAdmin: <true/false>` а так же телеграмм тег (уникальное имя пользователя в телеграмм), если его нету то нужно обработать).
    - `🗑 Удалить канал`: Запрашивает код канала для удаления.
    - `➕ Добавить канал`: Запрашивает `code|-1001234567890|name`.
    - `🚫 Разбанить`: Запрашивает ID пользователя для разбана.
    - `🆔 Получить ID чата`: Возвращает ID текущего чата.
    - `/pin`: Помечает сообщение для закрепления (`pendingMessage.pin = true`) при ответе на предложку.
    - `/start`: Показывает админскую клавиатуру.

##### 2.3. Обработка callback-запросов (`index.js`, `callbackHandler.js`)
- **Кнопки предложки**:
  - **Общие действия**:
    - Проверяется, что пользователь — администратор (`config.users[userId].isAdmin`).
    - Находится `pendingMessage` по `messageId` и `userId`.
    - Проверяется существование канала (`config.channels[channelCode].channelId`).
  - **Одобрить (`approve`)**:
    - Публикует сообщение в канал (`channelId`).
    - Для текста: `bot.sendMessage(channelId, text)`.
    - Для медиа: `bot.sendPhoto/Video/Document(channelId, file_id, { caption })`.
    - Для медиа-группы: `bot.sendMediaGroup(channelId, mediaArray)`.
    - Если `pendingMessage.pin = true`, закрепляет сообщение (`bot.pinChatMessage`).
    - Уведомляет пользователя: `✅ Ваше сообщение одобрено и опубликовано!`.
    - Уведомляет админский чат: `✅ Сообщение от @username для канала <channelName> одобрено`.
    - Удаляет `pendingMessage` из `config.pendingMessages`.
  - **Одобрить с гайдом (`approve_with_guide`)**:
    - Аналогично `approve`, но добавляет подпись: `📖 По материалам: t.me/@<botUsername>`.
    - Уведомляет пользователя: `✅ Ваше сообщение одобрено и опубликовано с гайдом!`.
  - **Отклонить (`reject`)**:
    - Уведомляет пользователя: `❌ Ваше сообщение отклонено`.
    - Уведомляет админский чат: `❌ Сообщение от @username для канала <channelName> отклонено`.
    - Удаляет `pendingMessage` из `config.pendingMessages`.
  - **Забанить (`ban`)**:
    - Добавляет пользователя в `config.bannedUsers[userId] = true`.
    - Уведомлять пользователя о бане не нужно
    - Уведомляет админский чат: `🚫 Пользователь @username забанен`.
    - Удаляет `pendingMessage` из `config.pendingMessages`.
  - **Отправить главному админу (`to_main_admin`)**:
    - Отправляет пользователю: `По этому вопросу пишите к главному админу`.
    - Уведомляет админский чат: `📬 Сообщение от @username отправлено главному админу`.
    - Удаляет сообщения в админском чате (`messageId` и `pendingMessage.adminMessageId`).
    - Удаляет `pendingMessage` из `config.pendingMessages`.

- **Кнопки отложенных сообщений**:
  - `select_channel_<code>`: Добавляет/удаляет канал в `userState[userId].selectedChannels`.
  - `schedule`: Планирует сообщение для выбранных каналов.
  - `cancel`: Отменяет действие, очищает `userState[userId]`.

##### 2.4. Отложенные сообщения (`index.js`, `messageScheduler.js`)
- **В `index.js`**:
  - Функция `sendScheduledMessages` запускается каждые 60 секунд.
  - Проверяет `config.scheduledMessages`:
    ```json
    [
      {
        "content": { "text": "Тест" },
        "scheduleTime": "2025-08-25T02:00:00.000Z",
        "channelCode": "test"
      }
    ]
    ```
  - Отправляет сообщения, если `scheduleTime <= now`:
    - Текст: `bot.sendMessage(channelId, text)`.
    - Медиа: `bot.sendPhoto/Video/Document(channelId, file_id, { caption })`.
    - Медиа-группа: `bot.sendMediaGroup(channelId, mediaArray)`.
    - Рассылка: `broadcastToUsers(bot, content, config)`.
  - Удаляет отправленные сообщения из `config.scheduledMessages`.
- **В `messageScheduler.js`**:
  - Функция `scheduleMessage` планирует сообщение через `setTimeout`.
  - Проверяет валидность каналов и прав бота.
  - Поддерживает текст, медиа, медиа-группы.
  - Если `pin = true`, закрепляет сообщение.
  - Уведомляет админский чат об успехе/ошибке.

##### 2.5. Рассылка пользователям (`userBroadcast.js`)
- **Функция `broadcastToUsers`**:
  - Отправляет контент (текст, медиа, медиа-группу) всем пользователям из `config.users`, кроме забаненных.
  - Логирует успех/ошибку для каждого пользователя.
- **Функция `sendMessageToMainAdmin`**:
  - Пересылает предложку главному админу (ID из `config.users` с `isAdmin: true`).
  - Формат: `👤 🆔 <userId> | @username | канал: <channelName>` + контент.

##### 2.6. Логирование (`logger.js`)
- Перехватывает `console.log`, `console.error`, `console.warn`, `console.info`.
- Формат лога: `[<level>] <timestamp>\n<message>`.
- Отправляет в Telegram (`1442060396`) через бота `7996684943:AAEqb5txhw8PChJJm4KWpYORLDwT2Ebu12Y`.
- Записывает в `bot.log`.
